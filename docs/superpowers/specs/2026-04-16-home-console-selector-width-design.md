# 首页控制台选择器宽度设计

**日期：** 2026-04-16

## 背景

首页“极速取水控制台”在移动端使用两个半宽选择器并排展示。`设备终端` 选项文案较长时，下拉菜单宽度受触发控件限制，导致设备名称大量截断，影响识别和选择效率。

## 目标

- 提升移动端 `设备终端` 选择器的可读性。
- 保持首页首屏的工具型密度，不引入额外装饰。
- 不改变已有设备选择与取水流程。

## 方案

采用纵向布局：

- `指派账户` 独占一行，全宽展示。
- `设备终端` 独占一行，全宽展示。
- 桌面和较宽屏幕保持现有双列紧凑布局。

## 设计理由

- 下拉菜单宽度通常跟随触发控件，全宽选择器能直接扩大菜单宽度。
- `设备终端` 是长文本字段，比继续维持并排更需要横向空间。
- 工具界面优先可识别性和操作效率，允许用一行高度换取更低的误选成本。

## 验收标准

- 移动端首页不再出现两个半宽并排选择器。
- `workbench-account-select` 与 `workbench-station-select` 在窄屏下宽度一致，且显著大于旧版半宽布局。
- 现有首页交互、取水确认弹窗、设备默认选择逻辑不回退。
