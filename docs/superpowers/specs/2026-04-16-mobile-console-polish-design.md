# 移动端工作台与账号页收口设计

**背景**

上一轮重构后的桌面视觉语言已经建立，但移动端首页与账号页仍存在明显的空间浪费和层级割裂：

- 首页顶部统计卡在窄屏下垂直堆叠，占用首屏。
- 取水按钮文案偏长，按钮主信息不聚焦。
- 账号页自动化区域仍保留标题容器，与顶部操作区割裂。

**目标**

在不改动核心信息结构的前提下，把移动端界面压缩为更紧凑、更统一的操作布局：

- 首页顶部两项统计始终在一行展示，并显著压低高度。
- 取水按钮只保留“图标 + 容量”。
- 账号页自动化不再单独成块，只保留 `签到`、`抽奖` 两个按钮，并与顶部操作按钮统一风格。

**设计决策**

## 首页统计

- 保留两项统计，不删除。
- 将当前大卡片改为紧凑双列信息卡。
- 缩小内边距、标题字号和数值字号，确保在 320 宽度下仍能同排显示。
- 数值允许自适应缩放，避免再次触发溢出。

## 首页取水区

- 保留现有三项选择：账户、设备列表、设备终端。
- 去掉额外解释性 helper 文案，减少视觉噪音。
- 两个取水按钮统一改为“水滴图标 + 容量值”，例如 `7.5L`、`15L`。

## 账号页操作区

- 顶部操作区整合为单一按钮带。
- 保留 `刷新`、`导入`、`新增` 三个账户操作按钮。
- 将 `签到`、`抽奖` 直接并入同一视觉组，不再显示“自动化”标题、容器或说明文案。
- 按钮高度、圆角、色彩层级与顶部操作按钮保持一致。

**验证要点**

- 320 宽度下首页无溢出，统计卡仍同排。
- 首页取水按钮不包含“取水”字样，但保留图标与容量。
- 账号页不再出现“自动化”文本，只保留两个对应按钮。
