# WaterNode 桌面端字体统一设计

## 目标

修复 WaterNode 在 Windows 桌面端出现的中文文本粗细不一致问题。

本次改动通过两件事解决根因：

- 引入项目自带中文字体，避免依赖系统默认字体栈。
- 收敛全局文本层级与字重，减少组件各自定义粗细导致的割裂感。

## 问题归因

- 当前应用没有配置 `fontFamily`，桌面端完全依赖系统字体回退。
- 页面内大量混用 `w500 / w600 / w700` 和不同 `textTheme`。
- Windows 对中文 fallback 字体的字重映射不稳定，导致同屏文字看起来有的偏黑、有的偏飘。

## 方案选择

### 方案 A：引入项目字体并统一主题

- 使用项目内置 `Noto Sans SC`。
- 在 `pubspec.yaml` 注册字体资源。
- 在应用主题里统一 `fontFamily` 和 `TextTheme`。
- 组件尽量依赖全局文本样式，不再局部硬编码字重。

优点：

- Android / Windows 观感更一致。
- 根因修复，不依赖用户本机字体环境。
- 后续新增页面更容易保持一致。

缺点：

- 仓库体积会增加。

### 方案 B：仅收敛主题，不引入字体

- 保留系统字体。
- 只在主题和组件层减少局部 `fontWeight`。

优点：

- 改动更小。

缺点：

- 无法彻底消除 Windows 字体回退差异。

## 最终方案

采用方案 A。

## 架构设计

- 新增一个独立主题文件，集中维护字体家族和 `TextTheme`。
- `WaterNodeApp` 改为使用统一主题工厂创建 `ThemeData`。
- 常见文本层级约定：
  - 正文 / 表单：`w400`
  - 标签 / 导航：`w500`
  - 分区标题：`w600`
  - 核心数值 / 品牌强调：`w700`

## 改动范围

- `pubspec.yaml`：注册字体资源。
- `fonts/`：新增 `Noto Sans SC` 字体文件。
- `lib/app/app.dart`：切换到统一主题。
- `lib/app/theme/`：新增主题定义文件。
- 文本组件：移除不必要的局部粗体设置，改用主题层级。

## 测试策略

- 新增主题测试，验证应用启动后使用了预期字体家族。
- 验证主题的关键字重分层：
  - `bodyMedium = w400`
  - `labelMedium = w500`
  - `titleSmall = w600`
  - `titleMedium / titleLarge = w700`
- 运行现有全量测试与 `flutter analyze`。

## 非目标

- 不改整体色彩系统。
- 不做新的桌面端 UI 重设计。
- 不引入多字体混排策略。
