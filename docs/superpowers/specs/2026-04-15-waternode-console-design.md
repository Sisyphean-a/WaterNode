# WaterNode 控制台壳层设计

## 目标

重构 WaterNode 主界面，使首页只承担信息展示和取水入口，其他能力
全部迁入可折叠左侧边栏，并保持 Android / Windows 双端统一体验。

## 当前范围

- 提供统一 `ConsoleShellPage` 壳层，窄屏使用 `Drawer`，宽屏使用可折叠
  侧栏。
- 首页仅保留账号统计、取水入口和运行快照，不再直接承载批量任务按钮。
- 新增 `任务中心 / 批量任务` 页面，承接批量打卡、积分抽取和执行日志。
- 保留 `设备中心 / 终端大厅`、`账号中心 / 凭证管理`、
  `账号中心 / 登录授权` 页面，并纳入同一壳层导航。
- 仓库目标平台收敛为 Android + Windows，移除未使用平台模板和噪音文件。

## 信息架构

1. `总览中心`
   - `首页概览`：查看统计、运行快照、进入取水能力。
2. `任务中心`
   - `批量任务`：执行批量打卡、批量积分抽取，查看完整日志。
3. `设备中心`
   - `终端大厅`：选择区域并下发取水指令。
4. `账号中心`
   - `凭证管理`：查看测试账号状态并刷新。
   - `登录授权`：发送验证码并新增凭证。

## 架构设计

- 路由层继续保留独立路径，但统一映射到 `ConsoleShellPage`，由壳层
  根据 `activeRoute` 切换内容区。
- `DashboardController` 继续负责账号统计、批量任务与日志聚合；
  首页和任务页共享同一控制器，避免重复状态源。
- `DeviceController`、`CredentialController`、`AuthController`
  维持原有职责，不在壳层内重新装配。
- 折叠态侧栏仅展示图标，但为导航项补充 `Tooltip` 与 `Semantics`，
  提升可用性和可访问性。

## 关键交互

1. 启动默认进入 `首页概览`。
2. 点击“立即取水”跳转到 `终端大厅`。
3. 点击侧栏“批量任务”进入任务页，执行批量打卡或积分抽取。
4. 凭证管理页点击“新增登录凭证”跳转到登录授权页。
5. 登录授权成功后回到凭证管理页。

## 测试策略

- Widget 测试覆盖首页轻量化约束、侧栏分组导航、任务页入口、
  账号页迁移，以及宽屏折叠态 tooltip 行为。
- 控制器测试继续覆盖账号状态刷新、批量任务执行与日志聚合。
- 静态检查和全量测试作为交付前必跑项。

## 非目标

- 不恢复首页上的批量任务按钮。
- 不恢复 `ios/`、`linux/`、`macos/`、`web/` 平台模板。
- 不接入真实 IoT 终端接口，只保留显式失败占位。
