# WaterNode Tool Workbench Design

## 背景

当前控制台更接近展示型后台，而不是高频操作工具：
- 顶部和卡片占据大量垂直空间
- 多处说明文案重复、信息价值低
- 页面模块依赖大块留白分隔，导致首屏有效信息密度偏低
- 侧栏切换使用整页路由替换，造成壳层与导航一起重建，交互感受生硬

用户要求按工具标准做破坏性重构，因此本次直接重做壳层与主要页面布局，不保留当前视觉骨架。

## 目标

1. 改为单壳层常驻工作台，侧栏不再随内容切换重建
2. 所有页面改为紧凑工具布局，首屏优先展示状态、操作和结果
3. 删除无意义的解释性文案和装饰性空白
4. 统一成工作台式视觉语言：轻标题、工具栏、状态条、数据面板、日志区

## 方案比较

### 方案 A：保留当前多路由壳层，仅压缩间距

- 优点：改动小
- 缺点：整页重建问题仍在，结构问题不解决

### 方案 B：保留多路由，但抽出共享侧栏

- 优点：能部分缓解导航重建
- 缺点：仍然绕着旧结构打补丁，页面层次仍旧松散

### 方案 C：单壳层常驻 + 主体区视图切换

- 优点：导航稳定、刷新感自然、最适合工具型工作台
- 缺点：需要同步改造页面入口和测试

## 选型

采用方案 C。

原因：
- 这是唯一能根治“侧栏跟着一起刷新”的方案
- 它天然适合桌面/工具界面
- 便于后续继续扩展更多工作区，而不是堆更多路由壳层

## 信息架构

- 左栏：品牌、全局导航、状态提示
- 顶栏：当前模块名、上下文摘要、主操作
- 主体：按模块切换，不重建壳层

页面定位：
- 首页：系统快照 + 快捷入口 + 最新日志
- 批量任务：操作面板 + 执行状态 + 完整日志
- 终端大厅：筛选工具栏 + 配置条 + 设备列表 + 操作结果
- 凭证管理：操作栏 + 汇总条 + 账号清单
- 登录授权：紧凑表单，不再用居中大卡片撑空间

## 视觉规范

- 不再使用大面积圆角卡片堆叠页面
- 模块间优先用细边框、浅底、分隔线，不用大空白制造层级
- 页面顶部采用 44-52px 左右的紧凑工具栏，而不是“标题 + 副标题 + 留白”
- 指标改为密集统计条，避免三张大卡片
- 列表项改为更像工作台条目，而不是居中的展示卡

## 交互原则

- 侧栏点击只切换主体内容
- 页面内跳转统一改为工作台切换，而非 `Get.offNamed`
- 保留窄屏 Drawer，但宽屏保持固定侧栏
- 高频操作靠近数据放置，不跨屏寻找

## 测试策略

- Widget 测试覆盖：单壳层导航切换、宽屏侧栏常驻、窄屏无溢出
- Widget 测试覆盖：首页和功能页新的工具栏/布局关键元素
- 全量 `flutter analyze` 与 `flutter test`
